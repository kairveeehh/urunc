    OverviewðŸ”—

    The urunc Workflow Dashboard has been redesigned to match the kata-containers style, providing a comprehensive view of CI/CD workflow status and job history.

    • Workflow Selector Dropdown - Switch between different workflows to view their job details
    • Job Search Filter - Search and filter jobs by name in real-time
    • Pass Rate Dashboard - View success rate percentage for each job
    • Run History Visualization - Last 10 runs displayed as color-coded status badges:

    • Dark Mode Support - Compatible with Material MkDocs theme

    Technical DetailsðŸ”—

    How It WorksðŸ”—

    1. Workflow Discovery
    2. Fetches all active workflows from GitHub API

    3. Populates dropdown selector with workflow names

    4. Job Aggregation

    5. When a workflow is selected, fetches the last 10 workflow runs
    6. For each run, retrieves all jobs and their statuses

    7. Aggregates job data across all runs

    8. Pass Rate Calculation

    9. Counts successful runs (conclusion: 'success')
    10. Calculates percentage: (passed / total) Ã— 100

    11. Displays jobs sorted by lowest pass rate first

    12. Search Filtering

    13. Client-side filtering of job names
    14. Real-time as user types
    15. Case-insensitive matching

    ComponentsðŸ”—

    File: docs/workflows.md

    Contains: - HTML structure with dropdown and search controls - CSS styling for dashboard layout and responsive design - JavaScript for API interaction and dynamic rendering

    Dependencies: - GitHub Actions REST API v3 - No external libraries required (vanilla JavaScript)

    UsageðŸ”—

    Accessing the DashboardðŸ”—

    1. Navigate to the urunc documentation site
    2. Click "CI Workflows" in the sidebar navigation
    3. The dashboard loads automatically at https://urunc.io/workflows

    Using the DashboardðŸ”—

    Select a Workflow: - Use the "Select Workflow" dropdown to choose which workflow to analyze - Dashboard updates automatically with that workflow's job information

    Search Jobs: - Type a job name in the "Search Jobs" field - Table filters in real-time as you type - Clear the search to see all jobs again

    View Job Details: - Job names are clickable links to GitHub Actions - Pass rate shows success percentage - Colored boxes show the status of the last 10 runs

    Performance ConsiderationsðŸ”—

    API CallsðŸ”—

    The dashboard makes the following API calls:

    1. Fetch Workflows (1 call) 

      GET /repos/urunc-dev/urunc/actions/workflows

    2. Fetch Workflow Runs (1 call per selected workflow) 

      GET /repos/urunc-dev/urunc/actions/workflows/{id}/runs?per_page=10

    3. Fetch Job Details (1 call per run, ~10 calls per workflow) 

      GET /repos/urunc-dev/urunc/actions/runs/{id}/jobs

    Total: ~12 API calls per workflow selection

    Rate LimitingðŸ”—

    GitHub API limits: - Unauthenticated: 60 requests/hour - Authenticated: 5,000 requests/hour

    The dashboard should work fine within these limits. Typical usage: - Initial page load: ~1-2 API calls - Per workflow selection: ~11-12 API calls

    • .jobs-table - Main table styling

    TroubleshootingðŸ”—

    Dashboard Not LoadingðŸ”—

    1. Check browser console (F12 â†’ Console tab)
    2. Verify GitHub API accessibility - try API URL directly
    3. Rate limit hit - wait 1 hour and reload (GitHub API rate limit resets hourly)
    4. Corporate firewall/proxy - may block GitHub API

    Empty ResultsðŸ”—

    • Ensure the selected workflow has runs in the last activity
    • Some workflows may not have recent executions
    • Try a different workflow from the dropdown

    Missing JobsðŸ”—

    • Jobs may be conditionally skipped in workflow
    • Run succeeded too quickly without job output
    • Check the workflow definition for conditional job steps
